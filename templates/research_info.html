COM748 Masters Research Project

About This Research Study

This platform is part of an academic dissertation investigating how automated feedback affects beginner programming skill development. Your participation directly contributes to evidence-based research in programming education.

Research Objective

This study investigates whether immediate, automated feedback accelerates beginner programming skill development compared to traditional minimal-feedback environments. The central question is: does feedback quality affect learning speed and problem-solving persistence in novice programmers?

Study Design

Two conditions are compared in this controlled study:

  • Static Mode (Control) — minimal binary feedback (Correct / Incorrect only)
  • Interactive Mode (Experimental) — detailed, automated error feedback and concept explanations

Participants self-select their preferred mode. This is intentional, allowing the research to observe self-selection behaviour alongside performance differences.

Privacy and Anonymity
  • No personal information is collected at any point
  • Every session is identified by a randomly generated ID
  • No names, emails, or device identifiers are stored
  • Data is stored on MongoDB Atlas (EU cloud region)
  • Participation can end at any time by closing the browser
  • Study falls under Category Z ethics (no formal consent form required)
Exercise Overview

Seven beginner Python exercises of gradually increasing complexity:

EX01 Variables and data assignment
EX02 Arithmetic operators
EX03 String methods
EX04 Conditional logic (if / elif / else)
EX05 While loops and iteration
EX06 For loops and lists
EX07 Functions (capstone task)

No prior programming experience required. Estimated time: 20 to 40 minutes.

Data Collected Per Attempt

Each code submission records exactly these fields:

session_id Randomly generated anonymous identifier
group_type static or interactive
exercise_id Which exercise was attempted
attempt_number Attempt count within session
result pass or fail
error_type Syntax, runtime, logic, or timeout
execution_time_ms Code run duration in milliseconds
timestamp UTC timestamp of each submission
How Data is Used

Anonymised attempt data is analysed to compute research metrics including pass rate comparisons, time-to-first-pass, persistence index, and error transition patterns. All findings are reported in aggregate form only. No individual session is ever identified or singled out in the research output.

Data may be included in the dissertation submission and may accompany future academic publications, always in anonymised aggregate form.
