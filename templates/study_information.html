COM748 Masters Research Project

Research Study Information

Complete information about this academic research study, study design, data collection practices, and your participation rights.

Research Objective

This study investigates whether immediate, automated feedback accelerates beginner programming skill development compared to traditional minimal-feedback environments.

The central question: Does feedback quality affect learning speed, problem-solving persistence, and understanding in novice programmers?

What You'll Do

Solve 7 beginner-friendly programming exercises (progressive complexity):

  • EX01–EX02: Variables, data types, operators
  • EX03–EX04: String methods, conditional logic
  • EX05–EX06: Loops, iteration, collections
  • EX07: Functions (capstone)

Available in Python and JavaScript. No prior experience required. Estimated time: 20–40 minutes.

Two Learning Modes

Static Mode (Control)

Minimal feedback: only "Correct" or "Incorrect" per submission.

Interactive Mode (Experimental)

Detailed feedback with error explanations, concept hints, and guided debugging.

You choose your mode. This intentional self-selection is part of the research design.

A/B Experiment Design

Within each learning mode, sessions are randomly assigned to one of two experiment arms:

Arm A (Control):

Standard learning flow. No adaptive recommendations.

Arm B (Adaptive):

Activity-based recommendations triggered by detected errors or difficulty patterns.

Assignment is randomized to enable fair comparison of recommendation effectiveness.

Why Recommendations Appear

If you're in Arm B, recommendations are triggered by:

  • Repeated failed attempts on the same task
  • Detected syntax or runtime errors
  • Logic mismatches indicating conceptual gaps
  • Topic completion patterns

Recommendations suggest remedial topics, next learning progressions, or conceptual review.

What Data is Collected

For each code submission, we record:

session_id Anonymized session identifier
group_type static or interactive
exercise_id, attempt_number Which exercise, which attempt
result pass or fail
error_type syntax, runtime, logic, timeout
execution_time_ms Code run duration
timestamp When submitted (UTC)

We also record quiz responses and which recommendations were shown.

Research Questions & Hypotheses

RQ1: Do adaptive recommendations improve pass rates?

RQ2: Which error patterns predict exercise difficulty?

RQ3: Does quiz performance correlate with coding success?

RQ4: Are JavaScript tasks harder than Python for beginners?

Analysis & Export

All data is analysed to compute:

  • Pass rates and attempt counts
  • Time-to-first-pass by topic and language
  • Error transition patterns
  • Persistence and completion metrics

Results are exported for statistical analysis in SPSS, R, Python, or Excel.

Privacy Guarantees
  • No personal information collected: No names, emails, or device IDs.
  • Anonymized sessions: Each session uses a randomly generated ID.
  • No tracking: Only exercise, quiz, and recommendation events logged.
  • Secure storage: Data stored in EU-region MongoDB Atlas with encryption.
  • Your control: Close the browser anytime to end participation.
Ethics & Regulatory Status

This study is approved for low-risk educational research.

  • — Category Z: No formal ethics consent form required (per university policy)
  • — Educational benefit clear: All feedback is pedagogically justified
  • — Participant agency: You choose mode and can exit anytime
  • — Transparent design: This page fully discloses what happens
Data Use & Publication

All data is reported in anonymised aggregate form only:

  • Included in dissertation submission
  • May accompany future academic publications
  • Never identified by individual session
  • Only aggregate statistics reported
